## Исправлено / добавлено

- **Отзывы на лендинге:** в блоке «Говорят те, кто проверил» цитата выводилась как `t.q` (undefined). Заменено на `tc.q`.
- **Кнопка «Смотреть демо»:** по клику — плавный скролл к блоку «Три шага» (#lprocess).
- **Лендинг на мобильных:** гамбургер-меню (иконка ☰), по клику — выезжающая панель с навигацией, языком и кнопками.
- **Поделиться картой:** кнопка «Поделиться» в тулбаре редактора. Генерируется ссылка вида `#share=xxx`, данные карты сохраняются в localStorage. По открытию ссылки — экран просмотра карты в режиме только чтение (без редактирования). Ссылка работает на том же устройстве/браузере; для шаринга между пользователями нужен бэкенд.
- **Роль «Зритель»:** при открытии карты из проекта проверяется роль пользователя (`myRole`). Если роль `viewer` — в MapEditor передаётся `readOnly={true}`: скрыты кнопки добавления шага, связывания, импорта, симуляции, шаблонов; перетаскивание узлов отключено; панель узла в режиме только просмотр.
- **Экспорт в PDF:** кнопка «⬇ PDF» в блоке экспорта. Открывается окно печати с таблицей шагов (название, статус, прогресс, дедлайн); пользователь может сохранить как PDF через «Печать → Сохранить как PDF».
- **Профиль на мобильных:** модалка профиля на узких экранах отображается как bottom sheet (прижата к низу, скругление сверху, max-height 90vh).
- **Удаление аккаунта:** вместо `window.confirm` — своя модалка с кнопками «Отмена» и «Удалить навсегда».
- **Даунгрейд тарифа:** при выборе тарифа ниже текущего показывается предупреждение: «При понижении тарифа часть возможностей будет ограничена (лимит карт, сценариев, участников).»

## Логика и UX

- **Лимиты при даунгрейде:** Если пользователь перешёл с Pro на Free, у него может остаться 5 карт при лимите 1. Сейчас при создании новой карты показывается «Лимит исчерпан» и выбор «заменить» или «улучшить тариф». Имеет смысл при смене тарифа на более низкий показывать предупреждение: «У вас N карт, на тарифе X разрешено M. Часть карт будет только для просмотра» или предложить выбрать, какие удалить/архивировать.
- **Ошибки сети / AI:** При падении `callAI` ошибка пробрасывается, но в части мест (например AI-панель) пользователь может не увидеть понятное сообщение. Проверить все вызовы AI и везде показывать toast/сообщение вида «Не удалось получить ответ. Проверьте сеть и ключ API».
- **Загрузка при первом открытии:** После сплэша/лендинга при переходе в «Проекты» данные берутся из хранилища. Если хранилище пустое или медленное — можно добавить скелетон или индикатор загрузки вместо пустого экрана.
- **Копирование/вставка узлов:** Есть Ctrl+C / Ctrl+V. При вставке узел дублируется с новым id — хорошо. Можно добавить подсказку в тултип или в блок шорткатов: «Выделите узел и нажмите Ctrl+C, затем Ctrl+V для копирования».

## Дизайн и консистентность

- **Мобильная версия:** Профиль на мобильных уже в виде bottom sheet. Остальные модалки (шаблоны, AI, тарифы при первом выборе) при желании тоже можно оформить как bottom sheet на узких экранах.
- **Лендинг — гамбургер-меню:** Реализовано: на ширине ≤640px показывается иконка ☰ и выезжающая панель.
- **Единые отступы и радиусы:** В проекте смешаны значения вроде 8, 9, 10, 11, 12 для border-radius и padding. Для полировки можно ввести константы (например `--radius-sm: 8px`, `--radius-md: 12px`) и использовать их в основных блоках.
- **Тема «Светлая»:** Переключатель есть, CSS-переменные для light заданы. Пробежаться по экранам (лендинг, проекты, редактор, профиль) и убедиться, что нет тёмных захардкоженных цветов (например `#0d1829`, `#070b14`) в текстах и фонах при светлой теме.
- **Тарифы на лендинге:** В PRICING на лендинге только Free, Pro, Enterprise. В приложении — Free, Starter, Pro, Team, Enterprise. Имеет смысл либо добавить Starter и Team на лендинг, либо оставить «топ-3» и написать «Полный список тарифов — после регистрации».
- **Иконки и эмодзи:** Часть кнопок с эмодзи (✦, 🗺, 📊), часть с текстом. Для консистентности можно выбрать один стиль для основных действий (например все основные CTA — текст + одна иконка) и придерживаться его в тулбаре и модалках.
